# 网上商城系统 —— 答辩演示脚本

## 一、项目概述（2 分钟）

### 项目定位
一个**多商户电商平台**，集成**大数据推荐系统**，使用 Hadoop + Spark 实现用户行为分析和个性化商品推送。

### 技术架构

```
┌──────────────────────────────────────────────────────────────────┐
│                    用户浏览器 (Next.js :8000)                      │
└─────────────────────────┬────────────────────────────────────────┘
                          │ HTTP/REST API
┌─────────────────────────▼────────────────────────────────────────┐
│                  FastAPI 后端 (:8001)                              │
│          JWT 认证 · 角色鉴权 · 行为事件采集                         │
└──────────┬─────────────────┬──────────────────┬──────────────────┘
           │                 │                  │
┌──────────▼──────┐  ┌───────▼───────┐  ┌───────▼──────────────┐
│  PostgreSQL 16  │  │  MinIO (S3)   │  │  大数据处理链路        │
│  业务数据库      │  │  商品图片存储  │  │                      │
└─────────────────┘  └───────────────┘  │  用户行为事件采集       │
                                         │    ↓                  │
                                         │  数据导出 → HDFS 上传  │
                                         │    ↓                  │
                                         │  Spark 推荐计算        │
                                         │    ↓                  │
                                         │  结果回写 PostgreSQL    │
                                         │    ↓                  │
                                         │  推荐 API 对外服务      │
                                         └──────────────────────┘
```

### 核心功能模块

| 模块 | 功能 |
|------|------|
| 用户系统 | 注册/登录（JWT）、三级角色（customer/manager/admin）、个人中心 |
| 商品浏览 | 商城大厅、分类浏览、搜索筛选、商品详情、多规格变体 |
| 购物车 | 游客购物车、登录用户购物车、增删改查 |
| 订单支付 | 下单结算、库存扣减、支付处理、订单确认 |
| 物流发货 | 发货管理、物流追踪、预计送达 |
| 多商户 | 店铺注册、审核上架、商户管理后台 |
| 管理后台 | 平台概览、用户/商品/订单/店铺管理 |
| **大数据推荐** | **行为事件采集、HDFS 存储、Spark 计算、个性化推荐** |

### 阶段 1：项目启动与环境（演示前提）

**演示前准备**：
```bash
docker compose up -d    # 启动所有服务（PostgreSQL、MinIO、FastAPI、Next.js、Hadoop、Spark）
```

**验证启动**：

- 终端执行 `curl http://localhost:8001/health` 确认后端正常

> **讲解要点**：项目使用 Docker Compose 一键部署，包含 7 个 docker-compose 文件，管理前端、后端、数据库、对象存储、Hadoop HDFS 集群（1 master + 2 worker）、Spark 集群（1 master + 1 worker）共 10+ 个服务。

### 阶段 2：用户认证与注册（3 分钟）

#### 2.1 用户注册
1. 访问 `/auth/register` 注册页面
2. 填写：用户名、邮箱、密码、姓名、手机号
3. 提交注册，展示注册成功

> **讲解要点**：密码使用 bcrypt 加密存储，账户数据写入 PostgreSQL `accounts` 表。

#### 2.2 用户登录
1. 访问 `/auth/login` 登录页面
2. 输入注册的邮箱和密码
3. 登录成功，展示 JWT token 存入 cookie

> **讲解要点**：JWT 使用 HS256 算法签名，有效期 24 小时。前端中间件根据 token 中的 role 字段实现路由级别的角色鉴权。

#### 2.4 登出功能
1. 在商城大厅（Hall）页面右上角，登录状态下显示「Account」和「Log out」按钮
2. 个人中心（Account）页面侧边栏底部、「Customer Panel」页面头部和侧边栏均有「Log out」按钮
3. Manager 和 Admin 后台同样有「Sign out」按钮
4. 点击登出按钮，系统执行：
   - 服务端删除 `shopping_token` cookie（httpOnly cookie 只能服务端删除）
   - 重定向到首页 `/`（未登录状态主界面）
5. **安全机制**：bfcache 守卫组件监听 `pageshow` 事件，当用户点击浏览器"返回"键回到已登出的页面时，检测到 cookie 已不存在则自动刷新页面，防止看到缓存的登录态界面

> **讲解要点**：所有登出统一使用 Server Action（`signout()`），服务端删除 httpOnly cookie + `redirect("/")`。bfcache 守卫确保返回上一页时不会看到残留的登录态，避免安全问题。登出覆盖商城大厅、分类浏览、店铺浏览、个人中心、Customer Panel、Manager Panel、Admin Panel 全部 7 个页面入口。

#### 2.3 三种角色演示
- **customer**：注册即获得，可浏览商品、下单购买
- **manager**：使用种子数据账号 `seed_mgr_nike` 登录，可管理店铺和商品
- **admin**：使用管理员账号登录，可进行平台级管理

> **讲解要点**：展示角色鉴权的实际效果——不同角色看到的导航菜单和可访问页面不同。未登录用户访问 `/admin` 或 `/manager` 会被中间件重定向到登录页。

### 阶段 10：大数据推荐系统（重点展示，8 分钟）

> **这是项目的核心亮点，需要详细讲解完整的数据处理链路。**

#### 10.1 用户行为事件采集

**演示**：打开浏览器开发者工具 Network 面板，浏览商品时观察事件上报。

讲解内容：
- 11 种行为事件类型：

| 事件类型 | 触发场景 | 权重 |
|----------|----------|------|
| `product_view` | 浏览商品详情 | 1 |
| `search` | 搜索商品 | 2 |
| `add_to_cart` | 加入购物车 | 5 |
| `remove_from_cart` | 移除购物车 | — |
| `checkout_start` | 开始结算 | — |
| `order_created` | 创建订单 | 8 |
| `order_paid` | 支付完成 | 10 |
| `favorite_product` | 收藏商品 | 6 |
| `recommendation_impression` | 推荐曝光 | — |
| `recommendation_click` | 推荐点击 | — |
| `recommendation_add_to_cart` | 推荐加购 | — |

- 同时支持已登录用户（account_id）和匿名用户（anonymous_id + session_id）追踪
- 事件包含丰富的上下文信息：商品 ID、分类、店铺、搜索词、来源页面等
- API：`POST /events`，fire-and-forget 模式，返回 202 Accepted

**数据库表**：`user_behavior_events`，所有行为事件持久化存储。

#### 10.2 事件数据导出

**演示**：


**输出目录结构**：
```

```

> **讲解要点**：导出脚本支持按日期范围增量导出，按 dt=YYYY-MM-DD 分区（Hive 分区规范），输出格式为 JSONL（每行一个 JSON 对象），适合大数据处理。

#### 10.3 数据上传到 HDFS

**演示**：
```bash
# 上传分区数据到 HDFS

```

**HDFS 目录结构**：
```

  dt=2026-07-13/

```

> **讲解要点**：HDFS 集群由 1 个 NameNode（master）+ 2 个 DataNode（slaver1、slaver2）组成，提供分布式、高可用的数据存储。

#### 10.4 Spark 推荐计算（核心算法）

**演示**：
```bash
# 运行完整推荐流水线
docker exec spark-master spark-submit \
  /scripts/run_all.py \
  --input /data/raw/events/ \
  --output /data/recommendations/
```

**三个计算任务逐一讲解**：

##### 任务一：热门商品计算（Popular Products）

**算法原理**：
```
score = SUM(事件类型权重) × exp(-事件距今天数 / 30)
```

- 不同类型行为赋予不同权重（购买 10 > 下单 8 > 收藏 6 > 加购 5 > 搜索 2 > 浏览 1）
- 引入**时间衰减因子** `exp(-days/30)`，确保近期热门商品排名更高
- 分别计算全站（home）和购物车（cart）两个场景

**输出表**：`popular_products`

##### 任务二：用户偏好画像（User Preferences）

**算法原理**：
- 按用户维度聚合行为数据
- 统计每个用户的：
  - **品类偏好**：对各个商品分类的兴趣分数
  - **店铺偏好**：对各个店铺的兴趣分数
  - **商品偏好**：对具体商品的兴趣分数
  - **价格区间**：浏览/购买商品的价格范围（min/max）
- 输出每个用户的 Top-N 偏好品类、店铺、商品

**输出表**：`recommendation_results`（scene=personalized）

##### 任务三：商品相似度计算（Item Similarity）

**算法原理**——基于协同过滤的共现矩阵：

```
                          Σ(用户对商品A的得分 × 用户对商品B的得分)
similarity(A,B) = ─────────────────────────────────────────────────
                  sqrt( Σ(用户对A的得分²) × Σ(用户对B的得分²) )
```

- 构建用户-商品评分矩阵
- 计算商品对之间的**余弦相似度**
- 对于每个商品，找出与其最相似的 Top-N 商品
- 只有被同一用户交互过的商品对才参与计算（共现条件）

**输出表**：`item_similarities`，每条记录包含 reason 字段（如 "Customers who viewed this also viewed"）

##### 管道编排（Pipeline Orchestrator）

`run_all.py` 按顺序执行三个任务：
1. `build_popular_products` → 热门商品
2. `build_user_preferences` → 用户画像
3. `build_item_similarity` → 商品相似度

支持两种运行模式：
- **local 模式**（开发/测试）：使用 Pandas + NumPy，无需 Spark 集群
- **spark 模式**（生产）：使用 PySpark，读写 HDFS

#### 10.5 推荐结果回写数据库

**演示**：
```bash
python scripts/import_recommendations.py \
  --input /data/recommendations/
```

将 Spark 输出的 JSONL 结果 upsert（插入或更新）到 PostgreSQL：
- `popular_products` 表
- `item_similarities` 表
- `recommendation_results` 表

> **讲解要点**：使用 PostgreSQL 的 `ON CONFLICT ... DO UPDATE` 实现幂等写入，重复运行不会产生重复数据。

#### 10.6 推荐 API 服务

**4 个推荐接口**：

| 接口 | 说明 | 数据来源 |
|------|------|----------|
| `GET /recommendations/home` | 首页热门推荐 | `popular_products` (scene=home) |
| `GET /recommendations/cart` | 购物车推荐 | `popular_products` (scene=cart)，排除购物车已有商品 |
| `GET /recommendations/products/{id}/similar` | 相似商品推荐 | `item_similarities` |
| `GET /recommendations/users/{username}` | 个性化推荐 | `recommendation_results` (scene=personalized) |

**前端展示效果**：
- 首页底部「热门推荐」区域
- 商品详情页「相似商品 / 看了又看」区域
- 购物车页面「你可能还喜欢」区域
- 个人中心「为你推荐」区域

**兜底策略**：
- 所有推荐结果过滤掉已下架、库存为 0 的商品
- 如果特定推荐结果不足，自动用热门商品补足
- 确保任何情况下用户都能看到推荐内容

#### 10.7 推荐效果闭环

**演示**：展示推荐反馈事件的采集：
- 用户在推荐区域看到商品 → `recommendation_impression`
- 用户点击推荐商品 → `recommendation_click`
- 用户从推荐将商品加入购物车 → `recommendation_add_to_cart`

> **讲解要点**：推荐系统的反馈数据会重新流入事件表，形成**数据闭环**——推荐曝光 → 用户交互 → 行为采集 → Spark 重算 → 推荐优化。这是一个持续迭代的推荐系统。

## 五、答辩重点提示

### 推荐系统的技术亮点

1. **完整的大数据链路**：行为采集 → PostgreSQL → 导出 → HDFS → Spark 计算 → 结果回写 → API 服务 → 前端展示，覆盖数据处理全流程

2. **三种推荐算法**：
   - **热门推荐**（加权 + 时间衰减）→ 解决冷启动
   - **协同过滤**（共现矩阵 + 余弦相似度）→ 商品关联
   - **用户画像**（多维度偏好聚合）→ 个性化推荐

3. **工程化设计**：
   - 双模式运行（local/spark），开发生产无缝切换
   - 兜底策略确保推荐永不空白
   - Hive 分区格式支持增量处理
   - 推荐反馈数据闭环

4. **前后端全栈**：推荐结果最终在前端多个页面位置呈现

### 可应对的常见答辩问题

| 问题 | 回答要点 |
|------|----------|
| 为什么用 Hadoop + Spark？ | 展示大数据处理能力；HDFS 分布式存储行为日志；Spark 内存计算高效处理用户行为矩阵 |
| 冷启动怎么解决？ | 热门商品推荐（全局统计）不依赖用户历史；新用户看到的是群体热门，随着行为积累逐步过渡到个性化 |
| 实时性如何？ | 当前是离线批处理架构；可以扩展为 Spark Streaming 或 Flink 实现实时推荐 |
| 推荐效果怎么评估？ | 当前通过推荐反馈事件闭环追踪（曝光→点击→加购），后续可加入 CTR、CVR 等指标 |
| 数据量大了怎么办？ | HDFS 天然支持横向扩展；Spark 可增加 worker 节点；分区表支持增量计算 |
